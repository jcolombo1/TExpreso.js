TExpreso

Liviano y veloz renderizador de contenidos a través de plantillas (templates), similar a Handlebars y Mustache, aunque adiciona variables runtime y otras mejoras para hacerle la vida más facil al "desarrollador front-end"

Download this project as a .zip file Download this project as a tar.gz file

A diferencia de Handlebars y Mustache, cuenta con variables de runtime, estructuras "if..else..end", además de auto-path del contexto de datos y bloques auto-condicionales y auto-iterativos. Es súmamente simple de usar y al igual que Handlebars permite embeber funciones "helpers" en runtime.

Incluye un plugin de jQuery (jquery.TExpreso.js) que facilita tanto la carga de archivos de templates y/o helpers externos, como también su renderización dentro de elementos del DOM.

Veamos un par de ejemplos de templates y datos para entender su funcionalidad...

Nota: Estos ejemplos están contenidos en el archivo "example.html" (obsérvelos en su explorador).

<p> {{saludo}} {{nombre}}! </p>

var datos = { nombre: "Jorge", saludo: "Buenos días" };

Las expresiones encerradas entre {{ }} son los elementos básicos a interpretar por el motor de plantillas TExpreso y que contienen los términos del "lenguaje de plantillas" que usted usará, entre ellos están las funciones helpers y las variables de datos. Por otro lado, todo lo que se encuentre por fuera de estas expresiones no será modificado respecto al código HTML resultante.

Como se observa en este ejemplo aterior, tanto {{nombre}} como {{saludo}}, son "variables", que al ser interpretadas por TExpreso con los datos suministrados, genera el HTML resultante con el saludo. Además note que tanto los tags <p> como el signo de admiración (!) no varían, debido a que no están dentro de expresiones {{ }}.

Si aún no lo hizo le propongo que ingrese ahora su nombre en el pequeño formulario que antecede a esto y pulse el botón "[ Interpretar ! ]", observará el efecto.


Bloques:

Un bloque es una estructura contenedora conformada por los marcadores
{{# expresion}} ... {{ end }}. De forma genérica se los ve así: 

{{# expresion}} 
	...contenido... 
{{end}}

Note que los bloques comienzan siempre con {{# }} y concluyen con {{ end }} .


Bloques iterativos:

Un bloque iterativo es un bloque contenedor que itera sobre el Array indicado en su expresión y que renderizará el código subyacente tantas veces como items contenga ese array. Veamos este ejemplo: 

{{# persons }}
	<p>{{firstName}} {{lastName}} vive en {{address.city}} </p> 
{{ else }}  
	<p>no hay personas en la lista!</p> 
{{ end }}

Note que en caso que el array no contega items, será el contenido subyacente a {{ else }} el renderizado ...

si lo renderizamos con el siguiente contexto de datos:

var myContext = {
  persons: [
    { firstName: 'Juan', lastName: 'Fulano', address: { city: 'Polvorines', province: 'BA' }  },
    { firstName: 'Ana', lastName: 'Sultano', address: { city: 'Sourdeaux', province: 'BA' }  },
  ]
};

... el resultado HTML será:

<p>Juan Fulano vive en Polvorines</p> 
<p>Ana Sultano vive en Sourdeaux</p>

en cambio, si lo renderizamos con el siguiente contexto de datos:

var myContext = { persons: [] };

... debido a que el array persons está vacío (o nulo/indefinido) se encaminará al {{ else }} :

<p>no hay personas en la lista!</p>

Si a este template algo más complejo:

<div>
	{{# if persons.length > 0}}
		<ul>
		{{# persons }}
	    	<li> <b>{{firstName}} {{lastName}}</b> vive en {{address.city}} </li> 
		{{ end }}  
	    </ul>
		<p>Hay {{persons.length}} personas</p>
	{{ else }}
		<p>No hay personas en la lista!</p>
	{{ end }}  
</div>

lo renderizamos con el siguiente contexto de datos:

var myContext = {
  persons: [
    { firstName: 'Juan', lastName: 'Fulano', address: { city: 'Polvorines', province: 'BA' }  },
    { firstName: 'Ana', lastName: 'Sultano', address: { city: 'Sourdeaux', province: 'BA' }  },
  ]
};

... el resultado HTML será:

<div>
  <ul>
      <li> <b>Juan Fulano</b> vive en Polvorines </li> 
      <li> <b>Ana Sultano</b> vive en Sourdeaux </li> 
  </ul>
  <p>Hay 2 personas</p>
</div>

Pero... si lo renderizamos con este otro contexto de datos:

var myContext = { persons: [] };  // "persons" no contiene elementos

... el resultado será entonces:

<div>
	<p>No hay personas!</p>
</div>
