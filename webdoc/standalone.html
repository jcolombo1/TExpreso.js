TExpreso

Modo Standalone

La forma de uso standalone podemos verla en el siguiente código javascript. No requiere dependencias.


<!doctype html>
<html>
	<head>
		<meta charset="utf-8" />
		<title>TExpreso - Standalone</title>
		<script src="js/TExpreso.min.js" type="text/javascript"></script>
	</head>
<body>

	<p id="view"> </p> 
 
<!-------------------- javascripts ----------------------->
 
<script type="text/javascript">
 
	var template = "{{saludo}} {{nombre}}!";
	window.TExpreso.add( 't1', template );
		
	var x = window.TExpreso.render('t1', {nombre:'Lucas', saludo:'Hola'});
	document.getElementById("view").innerHTML = x ;
 
</script>
 
</body>
</html>

Observe que en linea 17 se define una string con un pequeño Template. En linea 18 se le adiciona a TExpreso ese template asociado al nombre "t1". En linea 20 se invoca a renderizar el template "t1" con un contexto de datos, almacenando en la variable "x" el código HTML resultante. Por último se inserta en el elemento con id="view" ese contenido.

Lo complejo en esta modalidad es la carga de los templates, dado que como se ve, deben definirse en strings. Por el contrario, el plugin jquery.TExpreso falicita infinitamente la adquisición desde el DOM o desde URLs, además de la renderización directa sobre elementos HTML.
